结论：首页有较大优化空间

当前首页"信息够全，但呈现方式和转化引导偏弱"。最致命的两点是：整个 1 万多像素的长页面里没有一张真实的应用
截图，以及首屏之外完全没有下载入口。作为一款桌面应用的落地页，这是最大的机会损失。

  ---
当前主要问题

1. 看不到产品实物（最重要）
   首屏只有一个 App 图标，整个超长页面 0 张真实 UI
   截图。用户无法直观确认"装了之后能得到什么"，只能靠文字和图标网格来判断。

2. 下载按钮只出现在首屏
   页面这么长，Download 按钮却只在最顶部。滚动读完、意向最强的用户，最后看到的却是 Still Can't Find What You
   Need? → Let me know（反馈表单）作为结尾，收尾太弱。

3. 内容重复
   首屏的 4 张卡片（Native Speed / Offline AI / Universal Stack / Pro Utilities）和紧接着的 AppCanDo（同样 6
   项）诉求几乎一致，连着读很冗余。

4. 模块网格过大、单调
   Core + Tools 加起来约 80 个图标连续排列，把评价、GitHub 贡献者等社会证明挤到了很靠后的位置，滚动疲劳严重。

5. 内容未跟上最新版本
   网格里没看到最近新增的 RoadRunner / Swoole CLI / ZincSearch / Git / MkCert / SDKMAN / Cron
   Jobs（4.14.1～4.15.4 的新模块）。AppCanDo 里的 AI 模型名（Qwen 3.5 / Gemma 3 /
   GLM-5）也疑似超前于实际，需核实。

6. SEO/语义的小问题
   各区块标题用的是 <div class="...font-bold"> 而非 <h2>。虽然是 VitePress home 布局所限，但正文标题层级出现
   h1 →（无 h2），对 SEO 略有损失。
